## 💡 产品定位与痛点洞察
在传统的历史学研究与文献梳理中，研究者往往依赖“文档编号+Excel二维表”的模式进行史料长编，难以承载复杂的时间逻辑与叙事结构。
**ClioNote** 旨在将传统的史料长编过程抽象转化为**“时间序列知识建模”**，为高密度的文献研究提供极具秩序感的可视化交互体验。

## ✨ 核心产品特性 (Core Features)
* **时间轴可视化引擎**：打破传统二维表格限制，将分散的史料节点按绝对时间线进行串联与可视化渲染。
* **结构化节点卡片**：构建标准化的高密度信息卡片，支持史料来源、交叉验证备注及标签化管理。
* **大模型辅助录入 (Gemini API 驱动)**：利用底层 AI 能力，快速清洗、提取非结构化史料文本中的时间与实体信息，极大提升长编效率。

## 🛠 开发理念与敏捷工作流 (Vibe Coding)
本项目作为一个**概念验证 (POC) 原型**，摒弃了传统耗时的开发模式。
全栈代码（包含 React 前端架构与 Gemini API 联调）均采用 **Vibe Coding 工作流（AI 全栈代码生成与调试）** 在极短周期内落地。这一敏捷实践成功验证了“时间轴+核心节点”交互机制的产品可行性，沉淀了从“需求洞察”到“AI极速开发验证”的完整闭环经验。
